# 调研方案与结果分析

---

## 调研问卷

- 调研问卷信息全部来源于统一的“软件工程与计算大作业问卷”

## 调研结果分析

- 通过AI对调研问卷的高频词分析，以及对开放类问题的人工总结，对于核心问题的分析如下：

1. 核心功能需求：跑腿代取（快递/外卖）、咨询（学习/专业/培养方案）、二手交易、失物招领、资料分享.
   * 注：资料分享、校园资讯和失物招领板块在最重要的功能需求中的提及频率较高
2. 身份认证机制：大部分同学对于学号认证机制持赞同态度
3. 平台管理重点：内容审核（防止引战、违法信息）、言论规范、身份核验及信用体系维护
4. 收费机制：对于平台收费机制的统计中，虽然选择“完全免费”的人数最多，但是无法与其他选项形成明显差距，各个选项之间的差异也较小
5. 酬劳机制：大部分同学赞成平台提供积分机制，在小部分同学的反对意见中，希望直接用金钱交易
6. 线上交流：大部分同学对于线上交流功能持赞同态度
7. 任务成功判定：双方共同确认机制是高频出现的选项，且与其他选项形成明显差距

总体分析：通过对问卷的总体分析下，对与核心功能需求方面，我们可以将主要功能分割为两个板块，一个是“交易类”板块，包含跑腿代取、二手交易；另一个是
“信息类”板块，包含咨询和资料分享。而在身份认证机制方面，学号认证机制的高频出现说明了学生对于校园平台的整体封闭性的需求，倾向于校园内部的交流和交易环
境。在平台管理方面，其实可以参考市面上的流行平台的管理机制，主要是对于言论规范，内容审核的关注。而在于收费与酬劳机制方面，可以关注到学生的身份特征，
倾向于“免费”和“积分”制。但是回归到实际中，该校内平台的维护与运营成本较低，或许可以考虑完全免费的模式，可以依靠于校内的志愿者团队进行维护和运营。但是
对于酬劳方面，引入“积分”制的问题在于二手交易的场景中，积分的使用无法完全对应商品价值，可能会导致用户的参与积极性降低，因此如果要进行积分制设计，应该
考虑积分制的使用场景，是否可以于其他权益进行交换，或者实现多元化的货币体系。但是考虑到实现的复杂度，更倾向于使用“金钱”交易的方式，在交易类功能中，委托
方可以先展示期望价格，在后续的交易过程中，交易双方再进行议价，保证交易的自由度和灵活性。在于线上交流方面，大部分同学是支持线上交流功能的，但是我们必须
考虑平台的沟通功能于现有交流平台的替代性问题，平台的沟通功能需要解决现有平台的什么痛点，才能促进用户更倾向于平台内的沟通。最后在任务的成功判定方面，
问卷结果显示双方共同确认机制是被大多数同学认可的，但是再具体实现上需要考虑如何解决双方的信息不对称问题，以及如何保障公平性问题。

## 访谈提纲

- 访谈提纲基于上面的调研结果分析，针对一些宽泛的概念进行深入了解

1. 相对于学校提供的官方信息获取渠道，你希望在校园互助平台获得哪些信息？
   - 追问点：你最常遇到“信息找不到”或“信息太分散”的具体场景是什么？
   - 追问点：你希望信息以什么形式呈现（按学院分类、时间线、热门推荐、关键词搜索等）？
   - 追问点：你对信息时效性和准确性的最低要求是什么？
   - 记录要点：高频信息类型、使用场景、时效要求、可信度要求。

2. 如果互助平台在报酬方面采取积分制，这些积分除了用来发布任务之外，你希望还能用于什么场景？
   - 追问点：你更希望积分用于权益兑换、身份等级提升，还是服务优先权？
   - 追问点：什么样的积分规则会让你愿意长期参与（获取难度、消耗速度、有效期）？
   - 追问点：你认为积分是否应与信用体系联动（如违约扣分、优质服务加分）？
   - 记录要点：积分用途偏好、激励阈值、公平性顾虑、与信用体系关系。

3. 对于跑腿代取、咨询、二手交易、失物招领、资料分享这些功能，你认为可以如何分类，归为哪些板块？
   - 追问点：你更习惯按“需求类型”分类，还是按“紧急程度/交易属性”分类？
   - 追问点：哪些功能应该放在首页入口，哪些可以放在二级页面？
   - 追问点：哪些功能之间应互相跳转或联动（如失物招领与校园资讯）？
   - 记录要点：分类逻辑、优先级排序、首页入口建议、功能联动建议。

4. 相较于通过 QQ 等流行通讯工具进行直接沟通，你认为校园互助平台应具备哪些优势，才能促进持续使用？
   - 追问点：你认为“平台内沟通”最需要解决 QQ 无法解决的哪一类痛点？
   - 追问点：你更看重平台的匹配效率、身份可信、过程留痕，还是纠纷处理能力？
   - 追问点：在什么情况下你会选择平台沟通，而不是直接加联系方式私聊？
   - 记录要点：替代性优势、关键使用触发条件、平台独特价值。

5. 你认为对于这种小范围使用的买卖双方协商平台，应该如何保障“售后”机制？
   - 追问点：你认为应由平台提供哪些最基本售后保障（申诉入口、证据留存、仲裁规则）？
   - 追问点：平台在售后中应介入到什么程度（提醒、调解、强制裁定）？
   - 追问点：哪些违规行为应被重点约束，处罚方式应如何设计？
   - 记录要点：售后流程期望、平台介入边界、违规与处罚偏好。

## 访谈结果分析

- 访谈主要针对于调研结果中一些更深入的问题进行了信息收集以及讨论，分析结果如下：

1. 功能实现：在问卷结果分析中就已经提供了较为完善的实现思路，主要是分为两块，一个是“交易类”板块，包含跑腿代取、二手交易；另一个是“信息类”板块，包含咨询和资料分享。
而在访谈中，针对页内的信息分类问题，访谈对象都主要倾向于按照“需求类型”进行分类检索，同时也希望通过关键词搜索的方式进行信息的检索。我们在具体实现中则倾向于为每个信息
提供标签体系，用户可以通过标签进行信息的检索，同时也可以通过关键词搜索的方式进行信息检索。
2. 收费酬劳机制：平台收费机制的调查中，呈现出了多元化，差异小的特点，无法形成明显的倾向性，在访谈中发现，其实更多人倾向于免费与自愿的模式，实际上，平台的自愿捐助模式
其实和免费制并不冲突，因此在我们的具体实现中，可以为自愿捐助提供一个入口，用户可以通过该入口进行捐助，但是平台的核心功能仍然是完全免费的。同时在酬劳机制方面，虽然问卷
结果显示积分制的支持率较高，但是在访谈中发现，用户对于积分制的具体使用方面其实存在较大的分歧，倾向积分制的原因也是出于核心功能免费的诉求。因此在具体实现中，我们认为应
该将报酬的决策权交给用户，双方应当自行议价，但是为了保证交易信息的完整性与透明性，委托方在发布任务时需要展示期望价格，交易双方在后续的交易过程中再进行议价，保证交易的自由度和灵活性。
3. 线上交流：在问卷结果中，线上交流功能的支持率较高，但是在访谈中发现，用户对于平台内沟通功能的诉求其实处于一个较为模糊的态度，更重要的是能获取与对方进行沟通交流的渠道，
如果完全依赖于平台内的沟通功能，可能会存在沟通效率较低的问题，但是如果完全依赖于外部沟通工具，只提供联系方式，又涉及用户隐私保护的问题，因此在具体实现中，我们倾向于提供
简单的个人交流功能，用户可以通过平台内的沟通功能进行初步的沟通交流，在双方达成初步共识后，再提供联系方式进行后续的沟通交流。而用户暴露的公开联系方式仅限于邮箱地址，
且用户可以选择是否公开邮箱地址。
4. 任务判定：在访谈中，用户对于双方共同确认机制的认可度也较高，但是主要问题还是在售后方面，我们并没有专门使用人工售后的能力，人工智能实现或许可以作为后续扩展功能，但是在
目前产品周期中实现并不显示。考虑到实现的复杂度，我们倾向于大众参与的方式来处理售后投诉，用户可以通过平台提供的申诉入口进行投诉，平台会对投诉进行证据留存，并将投诉信息半公开，
让其他用户可以根据投诉信息进行投票，最终根据投票结果进行仲裁处理。同时对于一些违规行为，如恶意投诉、虚假信息等，平台会进行重点约束，处罚方式可以包括警告、禁言、封号等。
