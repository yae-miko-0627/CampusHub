# 用户故事

## 一 用户管理

1. 作为初次使用用户，我希望能够通过学号和密码注册，以便获得个人账户并使用平台的服务。
   * 验收标准：
   - 正常注册：输入有效学号并设置密码后，系统成功创建账户并提示注册成功。
   - 学号已存在：输入已注册的学号时，系统提示“该学号已注册”并阻止注册。
   - 密码强度：系统要求密码至少包含8个字符并包含两种及以上的字符，并提示用户设置更强密码以提高安全性。

2. 作为已注册用户，我希望能够通过学号和密码登录，以便访问我的个人账户和使用平台的功能。
   * 验收标准：
   - 正常登录：输入正确的学号和密码后，系统成功登录并跳转到用户主页。
   - 学号未注册：输入未注册的学号时，系统提示“该学号未注册”并阻止登录并提示请先注册用户。
   - 密码错误：输入错误的密码时，系统提示“密码错误”并允许重新输入。

3. 作为用户，我希望能够在个人中心管理我的个人信息（如头像、昵称、学院、联系方式、校区），以便保持我的账户信息更新和准确。
   * 验收标准：
   - 信息修改：用户可以成功修改个人信息并保存，系统提示“信息更新成功”。
   - 信息违规：审核用户的头像和昵称，确保其符合平台规范（如不含敏感词、图片不违规），并在审核通过后更新显示。

## 二 需求发布与浏览

1. 作为需求方，我希望能够发布需求，包含标题、描述、分类标签、地点、期限时间、报酬，以便让服务方了解我的需求并提供帮助。
   * 验收标准：
   - 发布成功：输入完整的需求信息后，系统成功发布需求并提示“发布成功”。
   - 信息验证：系统验证输入的信息是否完整且符合规范（标题和描述不含敏感词，报酬金额合理），并在验证失败时提示具体错误信息。

2. 作为服务方，我希望能够浏览需求列表，并通过分类标签、时间和距离等维度进行筛选和排序，以便快速找到符合我兴趣和能力的需求。
   * 验收标准：
   - 分类筛选：用户可以选择一个或多个分类标签进行筛选，系统正确显示符合条件的需求。

3. 作为服务方，我希望能够点击需求进入详情页，查看需求的详细信息和发布者的基本信息，以便评估是否接单。
   * 验收标准：
   - 详情展示：点击需求后，系统正确显示需求的标题、描述、分类、地点、期限时间、报酬以及发布者的头像、昵称、学院等基本信息。

## 三 订单流程管理

1. 作为服务方，我希望能够申请接单，并等待需求方的确认，以便开始提供服务。
   * 验收标准：
   - 接单申请：点击“申请接单”后，系统成功提交申请并提示“申请已提交，等待需求方确认”。
   - 接单确认：需求方收到接单申请通知，并在订单中心看到待确认的接单申请。

2. 作为需求方，我希望能够收到服务方的接单申请通知，并能够选择确认或拒绝，以便管理订单流程。
   * 验收标准：
   - 接单通知：当服务方申请接单后，需求方收到系统通知，并在订单中心看到待确认的接单申请。
   - 确认接单：点击“确认”后，系统将订单状态更新为“进行中”并提示“接单已确认”。
   - 拒绝接单：点击“拒绝”后，系统将订单状态保持为“待接单”并提示“接单已拒绝”。

3. 作为服务方，我希望能够完成互助任务后申请结项，并等待需求方的确认，以便完成订单流程。
   * 验收标准：
   - 申请结项：点击“申请结项”后，系统成功提交结项申请并提示“结项申请已提交，等待需求方确认”。
   - 结项确认：需求方收到结项申请通知，并在订单中心看到待确认的结项申请。
   - 确认结项：点击“确认完成”后，系统将订单状态更新为“待评价”并提示“订单已完成，等待评价”。
   - 自动确认：如果需求方在48小时内未确认结项，系统自动将订单状态更新为“待评价”并提示“订单已自动确认完成”。

## 四 评价与信用体系

1. 作为用户，我希望能够在订单完成后对交易对方进行双向评价，包括星级评分和文字评价，以便提供反馈并帮助其他用户做出决策。
   * 验收标准：
   - 评价提交：用户可以成功提交星级评分（1-5星）和文字评价，并提示“评价提交成功”。
   - 评价展示：提交的评价在用户主页和订单详情页正确显示。
   - 评价违规：系统审核评价内容，确保其符合平台规范（如不含敏感词、攻击性语言），并在审核通过后更新显示。

2. 作为用户，我希望能够查看自己的信用分和信用等级，以便了解自己的信誉状况。
   * 验收标准：
   - 信用展示：用户主页正确显示当前的信用分和对应的信用等级。
   - 信用更新：当用户提交评价后，系统根据评分计算新的信用分并更新显示。

## 五 论坛交流

1. 作为用户，我希望能在论坛寻找到自己感兴趣的信息，以便获取有用的资源和交流机会。
   * 验收标准：
   - 论坛浏览：用户可以成功进入论坛页面并浏览帖子列表。
   - 论坛搜索：用户可以通过关键词搜索帖子，系统正确显示符合条件的帖子。
   - 论坛分类：用户可以通过分类标签筛选帖子，系统正确显示符合条件的帖子。

2. 作为用户，我希望能够在论坛中发布帖子，分享信息或提问，以便与其他用户交流和获取帮助。
   * 验收标准：
   - 发布成功：输入完整的帖子信息后，系统成功发布帖子并提示“发布成功”。
   - 信息验证：系统验证输入的信息是否完整且符合规范（标题和内容不含敏感词），并在验证失败时提示具体错误信息。

3. 作为用户，我希望能够在论坛中回复帖子，与其他用户进行交流，以便分享观点和获取反馈。
   * 验收标准：
   - 回复展示：提交的回复在帖子详情页正确显示。
   - 回复违规：系统审核回复内容，确保其符合平台规范（如不含敏感词、攻击性语言），并在审核通过后更新显示。

## 六 消息通知与即时通讯

1. 作为用户，我希望能够收到系统通知，了解订单状态变更、接单申请等重要信息，以便及时响应和管理订单。
   * 验收标准：
   - 通知接收：当订单状态变更或有新的接单申请时，用户能够成功收到系统通知。
   - 通知展示：消息中心栏有红点提示，用户在消息中心正确看到所有未读和已读的系统通知。

2. 作为用户，我希望能够在平台内与交易对方进行即时通讯，以便沟通订单细节和解决问题。
   * 验收标准：
   - 即时通讯：用户可以成功进入与交易对方的聊天界面，并发送和接收消息。
   - 消息展示：发送的消息在聊天界面正确显示，并且对方能够及时收到。
   - 消息违规：系统审核聊天内容，确保其符合平台规范（如不含敏感词、攻击性语言），并在审核通过后更新显示。

## 七 投诉与纠纷处理

1. 作为用户，我希望能够在订单完成后提交投诉，以便对交易过程中遇到的问题进行反馈和寻求帮助。
   * 验收标准：
   - 投诉提交：用户可以成功提交投诉信息，并提示“投诉提交成功”。
   - 投诉处理：提交的投诉信息将公开在专门的纠纷处理页面，供高信用用户查看和参与处理，管理员也可介入处理相关纠纷，经过定时（48h）投票后，系统根据大众投票结果及管理员决策，给出纠纷处理结果，并对相关用户进行相应的信用调整以及后续的处理措施（如封禁、警告等）。
   - 投诉展示：提交的投诉在纠纷处理页面正确显示，并供高信用用户查看和参与处理。
   - 信息保护：投诉信息不应当包含敏感个人信息，系统应当对投诉内容进行审核和过滤。

2. 作为高信用用户，我希望能够查看和参与处理订单相关的投诉，以便维护平台的公平性和秩序。
   * 验收标准：
   - 投诉查看：高信用用户可以成功进入纠纷处理页面并查看订单相关的投诉信息。
   - 参与处理：高信用用户可以对投诉进行投票或提供意见，系统正确记录参与情况。

3. 作为管理员，我希望能够根据大众投票结果介入处理订单相关的投诉，以便维护平台的公平性和秩序。
    * 验收标准：
    - 投诉处理：管理员可以成功进入已有投票结果的投诉中，并查看投票信息。
    - 介入处理：管理员可以根据大众投票结果和自身判断，给出纠纷处理结果，并对相关用户进行相应的信用调整以及后续的处理措施（如封禁、警告等）。
